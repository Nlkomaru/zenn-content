# はじめに

今回[Tapo C210](https://www.tp-link.com/jp/smart-home/tapo/tapo-c210/)を用いて、睡眠検知システムを構築しました。
Tapo C210についてコストコにて、セールの際に2つで4000円程度で購入することができたため、これを用いて、睡眠検知を行うことを目的として、実装しました。

# 使用したもの
- Tapo C210
- MiniPC (GMKtec M5 Pro, CPU: 5700U, RAM: 40GB, SSD: 512GB) メモリは自分で増設
- 自分のゲーミングPC, NVIDIA RTX3070 (MiniPCとは別のPC)
- Google Colab

:::message
今回の記事について、自分自身は機械学習に関しては初心者であり、実装に伴い、LLMを用いたコードの生成や実装の方向性の相談などを行いながら実装を進めました。
:::

# 背景

背景として、去年(2025年)の4月頃から、健康管理を目的として、体重や体脂肪率、歩数、ストレス[^1]などを継続的に記録してきました。これらの既存のデータに加えて、睡眠の質も定量的に評価できれば、より健康管理の精度を高められると考えました。

当初は、この目的を達成するために[Pokémon Sleep](https://www.pokemonsleep.net)や[Sleep Cycle](https://sleepcycle.com/)を用いて、睡眠の質を定量的に評価することを試みました。睡眠分析の精度は申し分なく、睡眠の質を定量的に評価することはできましたが、これらのアプリは就寝前にスマートフォンを操作する必要があり、ある程度の操作も求められます。睡眠を目的としたアプリであるにもかかわらず、この操作が睡眠の妨げになっているように感じました。

そこで、睡眠前にスマートフォンを操作することなく、睡眠の質を定量的に評価できる方法を検討しました。結果として、[Withingsの睡眠トラッカー](https://www.withings.com/jp/ja/sleep)なども候補に挙がりましたが、学生の私には少し高価に感じられたため、より安価な方法を探すことにしました。

その結果、今回は格安の監視カメラを用いて睡眠検知を行うシステムを実装しました。

:::message
Withingsの睡眠トラッカーは約15,000円で購入でき、心拍数の計測やIFTTTとの連携なども可能なため、機能面ではこちらの方が優れているように感じます。
(実際には使用したことがないため何とも言えませんが もし利用している方がいれば、ぜひ感想を教えていただけるとうれしいです。)
:::

# 方針

はじめの方針としては、UltralyticsのYOLOv8を用いて、ベッドの上にいるかどうかを検出することを考えました。YOLOv8は物体検出の精度が高く、学習済みモデルも公開されているため、それを利用して検出をすることができると考えていました。
しかしながら、布団に入っている状態での姿勢の検出は、YOLOv8の学習済みモデルでは精度が低く、実用的なレベルで検出することは困難であることが分かりました。

そこで、次の方針として、単純な画像分類（Classification）を行うことを考えました。ベッド全体が映っている画像をそのまま学習させることで、睡眠中かどうかを判定できるのではないかと考えました。

分類にあたっては、以下の3つのクラスを設定しました。

- sleeping: 睡眠中である状態
- exist: 睡眠中ではないが、ベッドに人が存在する状態
- empty: ベッドに人が存在しない状態

これらのクラスを設定することで、睡眠中かどうかを判定できると考えました。

正直なところ、`exist`は不要ではないかとも思いました。しかし、どうせ取得できる情報であれば、多いに越したことはないだろうと考え、`exist`もクラスとして設定しました。

## 制約

先ほども述べたように、学生であるためあまり高価なクラウドサービスを利用することはできません。そのため、学習に当たっては、Google Colaboratoryを用いて、無料で学習を行うことにしました。
また、実際に分類を行う際には、家にあるMiniPCを用いて、学習済みモデルを用いた分類を行うことにしました。
したがって、CPUでの分類を行うことを前提として、学習済みモデルのサイズは小さくする必要がありました。


# 実装

## データセットの作成

まず初めに、データセットの構築を行うため、30日間、Tapo C210を用いて、ベッドの上を録画しました。録画についてはMiniPC上にあるK8sにて、ffmpeg経由でRTSPのストリームを取得し、MiniPC上に保存しました。録画した動画は、1日あたり約1.5GB程度の容量となりました。

その後、この画像をffmpegを用いて10分ごとに切り出し、`sleeping`, `exist`, `empty`の3つのクラスに分類しました。分類については、手動で行いました。
一日当たり、144枚の画像を分類する必要がありました。約30日分の画像を分類する必要がありましたが、一部似た画像については、はじくコードをCodexに書いてもらい、10%程減らすことができ、それらについてラベル付けを行いました。

ラベル付けについては、専用の[marimo notebook](https://marimo.io/)を用意し、自動で動画から画像へと分割、分割後の画像を表示し、それらについて私が、`sleeping`, `exist`, `empty`のどれなのかを選択することで、ラベル付けを行いました。

結果として、

## 学習の実施

学習に際して、日付ごとに、学習用と検証用に分割しました。train:validation:test = 6:2:2の割合で分割しました。
また、この分割したデータセットについて、train, val, testを入れ替えて、何度も学習を行い、精度の比較を行いました。
最終的には、テストケースを固定し、trainとvalを入れ替えて、何度も学習と評価を行い、精度の高いモデルを複数束ねて利用するという方向で進めました（詳細は後述の[アンサンブル](#アンサンブル)にて）。

回す回数をどれくらいと決めていなかったのですが、1回の学習あたり70回ほど学習を行いました。(試行錯誤で結果的には1000回くらい)
学習にあたっては、Google Colaboratoryの無料枠を用いたり、自身のPC(RTX3070, MiniPCとは別)を用いたりして、学習を行いました。

当初は、`mobilenetv3_small_100`を用いて、学習を行いましたが、実用的なレベルでの分類ではあるものの、もう少し精度を上げたいと考え、様々な学習済みモデルを試しました。その結果について以下に記録として残しておきます。

:::message alert
この表の時間とtest macro-f1については、すべて同じ学習データ、テストケースではあるものの、1ケースのみで比較しているため、著しく正確性を欠く可能性があります。あくまで参考程度にご覧ください。(Google Colab T4上での結果)
また、目安についてはChatGPTにて調査した内容を記載しています。正確性については保証できませんので、ご了承ください。

しかしながらこれらは、あまりにも学習に時間がかかりすぎるため、結果的には`hgnetv2_b2.ssld_stage2_ft_in1k`を用いて学習を行いました。

### 学習の設定

学習の細かい設定についても、記録として残しておきます。

- **エポック数**: 最大40エポックとし、検証データのmacro-F1が5エポック連続で改善しなければ早期終了(early stopping)するようにしました。
- **最適化**: AdamW(学習率 `4e-4`, weight decay `1e-4`)を使用しました。学習率は最初の2エポックでウォームアップし、その後はcosine annealingで徐々に減衰させました。
- **バッチサイズ**: 128
- **画像サイズ**: 224×224にリサイズして入力しました。
- **クラス重み**: `exist`のデータが他のクラスより少なかったため、クラスの出現頻度の逆数に応じた重みをlossに掛けることで、データの不均衡に対応しました。

また、データ拡張(data augmentation)として、ランダムなクロップ・拡大縮小(スケール0.3〜1.0)・アスペクト比変更・回転(±8度)・左右反転・明るさ/コントラスト/彩度のジッター・確率的なグレースケール化を適用しました。これらはCPUではなくGPU上でバッチごとにまとめて適用することで、CPUがボトルネックにならないようにしています。カメラの画角が寄り・引きで変わったり、昼夜で明るさが大きく変わったりするため、こうしたデータ拡張が精度に効いたように感じています。

このあたりのアイディアや実装についてはCodexに相談しながら進めました。

### アンサンブル

先述の通り、テストケースを固定したまま、trainとvalを入れ替えながら何度も学習を回すと、少しずつ異なる複数のモデルが手元に残ります。最終的には、これらのうち検証データのmacro-F1が高い上位5つのモデルを1つに束ね、それぞれの出力(softmax)を平均するアンサンブルモデルとして推論に利用しました。単一のモデルよりも予測が安定し、実際の推論もこのアンサンブルモデルで行っています。


## 結果

結果としては、`total: 540 / accuracy: 0.9204 / macro-F1: 0.9146`という結果を得ることができました。精度としては十分であり、実用的なレベルでの分類が可能であると考えています。
また、速度としても1枚の画像を解析するのに1秒かかっていない程度であるため、実用性としても十分であると考えています。

## 課題

基本的には満足しているのですが、完全に布団に入っている状態での分類は、`empty`と誤分類されることがあり、精度としてはまだ改善の余地があると考えています。今後は、機械学習ではなく、画像解析を用いたエッジ検出などを用いて、体動の有無を検出するなど別のアプローチも検討していきたいと考えています。
また、IKEAのサメをベッドの上に置いているのですが、これがあると、`exist`と誤分類されることがあり、精度の改善のためには、さらなるデータの追加なども並列で進めていきたいと考えています。
